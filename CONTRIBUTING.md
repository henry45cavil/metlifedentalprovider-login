# Contributing to MetlifeDentalProvider Login Expense Tracker

Thank you for your interest in contributing! This project aims to streamline financial tracking and expense management for dental care providers navigating portal workflows.

## Code of Conduct
We are committed to providing a welcoming, safe, and inclusive environment. All contributors are expected to conduct themselves professionally, treating everyone with respect, empathy, and constructive feedback. Harassment or discriminatory behavior of any kind will not be tolerated.

## How to Contribute
1. **Fork the Repository:** Create a personal copy of this project on your GitHub account.
2. **Clone Locally:** Pull the fork down to your local development environment.
3. **Create a Feature Branch:** Always work on a descriptive isolated branch (e.g., `feature/login-validation` or `bugfix/receipt-ui`).
4. **Implement & Test:** Make your changes, keeping coding styles clean and passing all tests.
5. **Commit & Push:** Write descriptive commit messages, then push to your fork.
6. **Open a Pull Request:** Target the `main` branch of this upstream repository.

## Reporting Issues
If you find a bug, operational quirk, or security vulnerability:
* Check the existing issues tab to ensure it hasn't been reported.
* Open a new issue using the standard Bug Report template.
* Provide clear reproduction steps, environment details (OS, Node version), and expected vs. actual behavior.

## Suggesting Features
Ideas for improvements are always welcome!
* Open a feature request issue.
* Explain the use case clearly: Who benefits from this feature? What problem does it solve within the provider platform context?

## Review Process
* All submissions require a pull request review before merging.
* A maintainer will review your code within 3–5 business days.
* Modifications may be requested to match architectural design or styling conventions.
